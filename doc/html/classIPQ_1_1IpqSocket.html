Detailed Description

A program's link to libipq.

Only one program can use libipq at a time.

Warning:
This class is not thread-safe.
Bug:

Connect() does not detect if other another program is using libipq. Instead, if another program is connected, recvPacket() will throw an exception. This is due to a limitation in libipq.

Due to a bug in Netfilter, changing the source IP address of a packet to a non-local address and then setting a verdict of NF_ACCEPT will cause it to be dropped. This has been observed when the packet was queued from the OUTPUT chain of the mangle table. A work-around is to set a verdict of NF_STOP instead of NF_ACCEPT, but be aware that this results in the packet being treated differently by iptables. Kernels prior to 2.6.12 do not have NF_STOP; defining NO_NF_STOP enables another work-around: all packets are immediately dropped, and packets that are to be accepted are re-injected through raw sockets. In this case, care must be taken to ensure that infinite queueing loops do not occur. Also, this work-around only works for IP packets.

IpqPacket * IPQ::IpqSocket::recvPacket ( bool  noblock = false) throw (IpqException)

Receive a packet from libipq.

Parameters:
noblockIf true, throw an IpqException if no packet is available. If false, block until a packet becomes available. (default: false)
Returns:
A dynamically-allocated packet object, which may be an IpqPacket or any of its subclasses. It must be freed using delete, and a verdict should be set on it using IpqSocket::sendResponse().
Exceptions:
IpqExceptionIf there was an error reading from the socket, no packet was immediately available and noblock was set, or a libipq error message was received.
Note:
If noblock was not set, then this method may block indefinately.
If NO_NF_STOP is defined, the packet is immediately dropped. It may be re-injected using IpqSocket::sendResponse().

Sets the verdict on a packet.

If the packet was changed, the original packet will be replaced with the modified version. If a verdict was already set on pkt, no action is taken beyond updating the packet checksums.

Parameters:
pktThe packet whose verdict to set.
vThe verdict to set.
Exceptions:
IpqExceptionIf the verdict is invalid or there was an error setting the verdict.
